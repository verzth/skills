# HTML Template for em-thinking Artifacts

Self-contained HTML template to render em-thinking artifacts as a human-readable review version.

---

## Used By

- **`/em-plan`** → renders `edd.html` (companion to `edd.md`)
- **`/em-review` Mode A** → renders `pr-review-<sha>.html` (companion to `pr-review-<sha>.md`)

Other skills (`/em-works`, `/em-review` Mode B) currently emit `.md` only. Future v1.5+ may extend coverage.

---

## Why Dual Output (.md + .html)

| File | Audience | Purpose |
|------|----------|---------|
| `edd.md` | Engineer (handoff), version control, future skill consumption | Source-of-truth, editable, diff-friendly |
| `edd.html` | EM, stakeholders, human reviewer | Self-contained, styled, print-friendly, shareable |

**Both must be written per skill invocation.** Content must be 1:1 consistent (same data, different rendering).

## Section Rendering Conventions

### Risk tier badge inline

```html
<span class="badge risk-t0">T0 — Critical</span>
<span class="badge risk-t1">T1 — High</span>
<span class="badge risk-t2">T2 — Standard</span>
<span class="badge risk-t3">T3 — Trivial</span>
```

Use in header meta + elsewhere in body if needed for inline mention.

### Severity badge

```html
<span class="badge severity-block">Block</span>
<span class="badge severity-major">Major</span>
<span class="badge severity-minor">Minor</span>
<span class="badge severity-info">Info</span>
```

### TL;DR card (top of artifact)

```html
<div class="tldr-card">
  <h2>TL;DR</h2>
  <p><strong>Problem:</strong> ...</p>
  <p><strong>Architecture summary:</strong> ...</p>
  <p><strong>Critical invariants:</strong> ...</p>
  <p><strong>Critical gaps:</strong> 0 (must resolve before em-works)</p>
  <p><strong>Recommended path:</strong> proceed to /em-works</p>
</div>
```

### ASCII diagram (default for all diagram types)

```html
<pre class="ascii-diagram">
┌─────────────────────┐
│   PaymentService    │
│  ┌───────────────┐  │
│  │  Idempotency  │  │
│  │     Cache     │  │
│  └───────────────┘  │
└──────────┬──────────┘
           │ calls
           ▼
┌─────────────────────┐
│ PaymentProviderAPI  │
└─────────────────────┘
</pre>
```

### Component diagram (HTML/CSS — optional alternative for component boundary)

Use if the component diagram is a simple linear flow and benefits from color coding. For complex diagrams (multi-arrow, branching), stick with ASCII.

```html
<div class="component-diagram">
  <div class="row">
    <div class="component" data-type="service">PaymentService</div>
    <span class="arrow">→</span>
    <span class="arrow-label">calls</span>
    <div class="component" data-type="external">PaymentProviderAPI</div>
  </div>
  <div class="row">
    <div class="component" data-type="service">PaymentService</div>
    <span class="arrow">→</span>
    <span class="arrow-label">writes</span>
    <div class="component" data-type="database">payments_db</div>
  </div>
  <div class="row">
    <div class="component" data-type="service">PaymentService</div>
    <span class="arrow">→</span>
    <span class="arrow-label">enqueue</span>
    <div class="component" data-type="queue">payment_events</div>
  </div>
</div>
```

`data-type` options: `service`, `external`, `database`, `queue`, `cache`.

```html

  <thead>
    <tr>
      <th>#</th>
      <th>Scenario</th>
      <th>Test?</th>

      <th>Severity</th>

  <tbody>

      <td>✓ retry</td>
      <td>Loud retry message</td>
      <td><span class="badge severity-major">Major</span></td>
    </tr>
    <tr class="critical-gap">
```

### Forcing question callout

```html
<div class="forcing-question">
  <strong>Q1 (Risk tier):</strong> Do you accept this as T0 or T1?
  <ul>
    <li>a) T0 (irreversible op, sensitive data)</li>
    <li>b) T1 (schema migration, breaking API)</li>
    <li>c) T2 (single-module additive)</li>
  </ul>
</div>
```

### Anti-pattern bullet list

```html
<ul>
  <li><div class="anti-pattern">Skip risk tier — leaves downstream without a proper gate.</div></li>
  <li><div class="anti-pattern">Jumping to implementation without state diagnosis.</div></li>
</ul>
```

```html

    <tr><td>...</td><td>...</td></tr>
```

## TOC Auto-Generation

Build from `<h2>` (top-level) and `<h3>` (nested). Every heading must have an `id` (slug from heading text).

Example:

```html
<nav class="toc" aria-label="Table of contents">
  <h2>Contents</h2>
  <ol>
    <li><a href="#tldr">TL;DR</a></li>
    <li><a href="#scope">Step-0 Scope Findings</a></li>
    <li>
      <a href="#architecture">Architecture</a>
      <ol>
        <li><a href="#component-boundaries">Component boundaries</a></li>
        <li><a href="#data-flow">Data flow</a></li>
        <li><a href="#trust-boundaries">Trust boundaries</a></li>
      </ol>
    </li>
    <li><a href="#invariants">Invariants</a></li>
    <li><a href="#failure-modes">Failure Modes</a></li>
    <li><a href="#test-strategy">Test Strategy</a></li>
    <li><a href="#handoff">Handoff</a></li>
  </ol>
</nav>
```

Heading with matching id:

```html
<h2 id="tldr">TL;DR</h2>
```

Slug rule: lowercase, replace space with `-`, strip non-alphanumeric except `-`.

## Common Mistakes

- ❌ **Skip HTML output** — user explicitly reviews via HTML, mandatory.
- ❌ **External CDN** (e.g. `<link href="https://cdn...">`) — breaks file:// loading + offline.
- ❌ **Mismatch .md ↔ .html content** — must be 1:1. HTML is rendering, not different content.
- ❌ **JavaScript dependencies** — not allowed, HTML must be pure static.
- ❌ **Hardcoded inline styles per element** — all styling via class. CSS already inlined in `<style>`.
- ❌ **Missing TOC** — required even for short artifacts (3-5 sections).
- ❌ **Critical gap row without class="critical-gap"** — loses visual signal.
- ❌ **Risk tier mention without `<span class="badge risk-tX">`** — loses color coding.

## File Output Convention

Same directory:
```
<working-dir>/
├── edd.md         ← source markdown
└── edd.html       ← rendered review version
```

Filenames must match (just different extensions). Never produce only one — always pair.

For PR reviews:
```
<working-dir>/
├── pr-review-<sha>.md
└── pr-review-<sha>.html
```

`<sha>` is short SHA (7 chars) of the PR head commit.
